# セキュリティポリシー

## 対象

現在保守するのは`1.x`系です。公開済み最新版で再現する問題を優先して確認します。

このゲームは静的ファイルだけで動作し、バックエンド、ログイン、分析、広告、課金、外部APIを使用しません。入力したプレイヤー名、物語進行、設定、収集記録はブラウザの`localStorage`へ保存します。

## 脆弱性を報告する

セキュリティ上の問題は、公開Issueへ実証コードや個人データを書かず、リポジトリホストのPrivate vulnerability reporting / Security advisoryが利用できる場合はそこから報告してください。非公開窓口が利用できない場合は、再現に必要な最小情報だけをIssueで連絡し、詳細の共有方法を管理者と調整してください。

報告には次を含めてください。

- 影響を受けるバージョンまたはコミット
- ブラウザ、OS、端末種別
- 再現手順と期待結果・実際の結果
- 想定する影響
- 回避策が分かる場合はその内容

セーブJSONには入力したプレイヤー名が含まれます。報告へ添付する前に、名前を架空の値へ置き換えるか、同じ手順で作成した検証用セーブを使用してください。秘密情報、Cookie、ほかのサイトのデータ、個人を特定できる情報は送らないでください。

## セキュリティ設計

- ユーザー入力をReactのテキストとして描画し、HTMLとして挿入しません。
- 名前は前後空白を除去し、Unicode単位で1〜12文字、制御文字なしを検証します。
- 読み込むセーブJSONはZodスキーマ、スキーマバージョン、コンテンツ参照で検証します。
- 自動・手動保存の書き換え前に、有効な直前データをバックアップします。
- 位置情報、カメラ、マイク、決済、USBをPermissions Policyで無効化します。
- Content Security Policyは同一オリジンの静的ファイルを基本とし、`object-src`と外部接続を禁止します。
- Service Workerは同一オリジンのGETだけを処理し、古いバージョンのキャッシュをactivation時に削除します。
- `.env*`、ログ、生成テスト結果はGit管理から除外します。

公開先で`public/_headers`が反映されることを、ブラウザのNetworkパネルまたはレスポンスヘッダーで確認してください。ホスティングサービスがこの形式に対応しない場合は、同等のヘッダーをサービス側で設定します。

## 依存パッケージ

依存更新前後に次を実行します。

```bash
npm ci
npm audit
npm run quality
npm run test:e2e
```

自動修正を無条件に適用せず、`package-lock.json`の差分、破壊的変更、ブラウザバンドルへ含まれるか、実際の到達可能性を確認します。ライセンス変更も[`THIRD_PARTY_NOTICES.md`](./THIRD_PARTY_NOTICES.md)へ反映します。

## 利用者向けの注意

- 信頼できる公開元からだけゲームを開き、セーブJSONも自分で書き出したものか信頼できる相手のものだけを読み込んでください。
- 共有端末ではプレイヤー名と物語記録が同じブラウザ利用者から見える可能性があります。利用後はゲーム内の「記録を初期化」またはブラウザのサイトデータ削除を使用してください。
- セーブは暗号化していません。実名や秘密情報をプレイヤー名へ入力しないことを推奨します。
- オフラインキャッシュと`localStorage`はブラウザのサイトデータ削除で消えます。必要な記録はJSONとして保管してください。
