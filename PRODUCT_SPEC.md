# 製品仕様：月影町 ― 十三番目の回覧板

## 1. 製品定義

### 一文

夕暮れの架空の町へ転入したプレイヤーが、町を歩き、住民と交流し、自室・郵便受け・掲示板・回覧板を通して「存在しない十三番目のお知らせ」の謎を追う、ブラウザ向け2.5D探索アドベンチャー。

### 体験の流れ

1. 到着：知らない終点の町へ降り立つ
2. 所属：転入届、住民票、自室、郵便受けを得る
3. 親密：住民の日常と小さな依頼に関わる
4. 違和感：部屋番号、人口、回覧板、町の記録が一致しない
5. 探索：町を歩き、会話と資料から手掛かりをつなぐ
6. 選択：真実を町へ返すか、外へ持ち出すか、役目を引き継ぐか決める
7. 余韻：クリア後も月影町が続いていると感じる

### 対象と長さ

- PC・スマートフォンのブラウザ利用者
- 物語、雰囲気、軽いミステリーが好きな人
- 初回20〜45分
- 全エンディング60〜120分
- 戦闘なし、ジャンプスケアなし、流血なし

## 2. 完成体験

次が一本につながっていること。

1. 古い本または封筒を開くタイトル演出
2. プレイヤー名を入力して転入届を作成
3. 月影町駅へ到着
4. 役場で住民票と月影荘203号室の鍵を受け取る
5. 3Dの町を歩く
6. 自室と郵便受けを確認する
7. 住民と会話し、小さな依頼を解決する
8. 朝・夕方・夜、7日間を進める
9. 日ごとに更新される手紙、掲示板、回覧板を読む
10. 十三番目のお知らせの手掛かりを10個以上集める
11. 放送塔で最終選択を行う
12. 3種類以上のエンディングへ到達
13. クリア後に住民票、記録、手紙、人物、実績を閲覧
14. 新規開始、続きから、最終日前から、セーブ書出・読込が可能

## 3. エリア

必須エリア：

1. 月影町駅
2. 町役場
3. 月影通り
4. 町内掲示板
5. 古書店「灯下書房」
6. 喫茶「宵待」
7. 月影荘
8. 自室203号室
9. 放送塔の丘

方式：

- 町のハブは小規模で密度のある3D空間
- 建物内部は独立シーン
- 屋内は2.5D、簡素な3D、立体的HTML UIのいずれでもよい
- 広さより、ランドマーク、生活感、調査対象、演出密度を優先
- 移動不能やカメラ事故を避けるため、単純な境界・コライダーを使う
- ナビメッシュが不安定なら、移動ノードやタップ可能範囲方式へ切り替える

## 4. 操作

### PC

- WASD／矢印：移動
- Enter／Space／E：決定・調査
- Escape：戻る・メニュー
- M：地図
- J：日記
- Tab：UIフォーカス

### モバイル

- 左下：仮想スティックまたはタップ移動
- 右下：調査
- UI表示中は3D操作を止める
- セーフエリア対応
- 縦横の両方で致命的に崩れない
- 片手操作を考慮する

### カメラ

- 半固定の斜め上視点
- プレイヤー追従
- 回転可能にする場合は角度制限
- 急な揺れ、壁へのめり込み、酔いやすい加速を避ける
- モーション軽減設定を反映する

## 5. 必須機能

### タイトル

- はじめから
- つづきから
- 記録
- 設定
- 演出スキップ
- `prefers-reduced-motion`対応

### 転入届・住民票

- プレイヤー名1〜12文字程度
- 日本語、記号、絵文字で壊れない
- HTML注入を無害化
- 会話、手紙、住民票へ名前反映
- 住所「月影町 月影荘203号室」
- 住民番号、転入日、印章風装飾
- 進行に応じて備考欄が変化
- 印刷または画像保存が可能なら実装

### 会話

- 名前、立ち絵または抽象的ポートレート、本文、選択肢
- テキスト送り、履歴、文字速度、自動送り、既読スキップ
- 条件分岐、信頼フラグ
- 一つの会話ボックスは原則80文字程度
- 内容はデータ駆動
- 重要情報は音声だけに依存しない

### 調査

- 調査対象への反応
- 近づいた時のラベル
- 日・時間・フラグで文言変化
- 重要な発見は日記へ記録
- 操作対象の重なりに対処

### 手紙・郵便受け

- 日ごとに手紙
- 未読表示
- 差出人、日付、件名、本文
- 条件付き返信または行動
- 差出人不明の手紙
- 自室から再読
- 最低7通＋エンディング差分

### 掲示板・回覧板

- 公式のお知らせ12種類
- 条件を満たすと13件目
- 日ごとに追加、剥離、改変
- 前に読んだ内容との差分が分かる
- 異常を色だけで示さない

### 日記・クエスト

- 現在の目的
- 完了済み出来事
- 人物メモ
- 手掛かり一覧
- 回覧板・手紙の収集
- 段階式ヒント
- 次の行動が分からなくならない

### セーブ

- localStorageまたはIndexedDB
- 自動・手動セーブ
- JSON書き出し・読み込み
- スキーマバージョン
- マイグレーション
- 破損時バックアップ復元
- リセット二段階確認
- 外部へ個人情報送信なし

### 設定

- BGM・効果音音量、ミュート
- 文字速度、自動送り
- モーション軽減
- 画質：自動・低・中・高
- 操作説明
- データ書出・読込・初期化
- 設定を保存

### 記録

- 到達エンディング
- 手紙、回覧板、手掛かり
- 人物プロフィール
- プレイ時間
- 実績6件以上
- 未発見はネタバレを避ける

### PWA・公開

- Cloudflare Pagesで静的公開可能
- SPAフォールバック
- Manifestとアイコン
- 更新時に古いキャッシュで壊れない
- オフライン時にタイトルと保存済み記録を最低限閲覧可能
- OGP、title、description、robots.txt、sitemap.xml
- プライバシー説明

## 6. 技術仕様

推奨スタック。既存コードや検証結果から合理的理由があれば変更可。

- Node.js LTS
- npm
- React
- TypeScript
- Vite
- Three.js
- React Three Fiber
- `@react-three/drei`
- Zustand
- Zod
- React Routerまたは状態ベースルーティング
- Framer Motion
- Vitest
- Testing Library
- Playwright
- ESLint
- Prettier

音声：

- Web Audio APIによる生成を優先
- またはライセンス明確な小容量素材
- 自動再生制限を正しく扱う
- 音なしでも情報が欠落しない

推奨構成：

```text
src/
  app/
  game/
    engine/
    state/
    events/
    quests/
    dialogue/
    save/
    time/
  scenes/
  world/
  ui/
  content/
    characters/
    dialogue/
    letters/
    bulletins/
    quests/
    endings/
  audio/
  assets/
  styles/
  tests/
e2e/
public/
docs/
artifacts/screenshots/
```

設計要件：

- ゲーム状態とUI状態を分離
- 物語イベントを「条件」と「効果」のデータで表現
- 型付きIDまたは集中定義されたIDを使う
- セーブ対象と一時状態を分離
- 3D表示詳細を物語ロジックへ漏らさない
- コンテンツ整合性検証スクリプトを作る
- 未定義の人物、会話、フラグ、クエスト参照を検出
- 可能ならヘッドレスな状態シミュレータで全日・全エンディング到達性を検証

## 7. 性能

- 初期表示を真っ白にしない
- ローディング状態を表示
- 一般的スマートフォンで30fpsを目標
- devicePixelRatioに上限
- 自動画質判定
- 低画質では影、粒子、ポスト処理を削減
- 重い素材を遅延読込
- 非表示タブで描画負荷を抑制
- シーン再入場でオブジェクトやイベントを重複させない
- WebGL非対応時に説明、再試行、低画質案内
- バンドルサイズと性能上の判断を`DOCUMENTATION.md`へ記録

## 8. セキュリティ・プライバシー

- `dangerouslySetInnerHTML`を原則禁止
- セーブ読込をZod等で検証
- 秘密情報を置かない
- 外部URLは安全な属性
- 分析、広告、トラッキングなし
- 依存脆弱性を確認し、重大問題を放置しない
- ゲームは原則ローカル完結

## 11. 必須完成後の改善ループ

必須条件を満たした後も余力があれば、機能を無秩序に増やさず、以下を繰り返す。

1. 全品質コマンドとE2Eを実行
2. デスクトップとモバイルのスクリーンショットを確認
3. 次を100点満点で採点
   - 完成度
   - 初見理解
   - 操作性
   - 視覚的一貫性
   - 物語の理解度
   - モバイル品質
   - 性能
   - アクセシビリティ
   - テスト信頼性
   - 保守性
4. 最低点の軸を一つ選ぶ
5. その軸だけを改善
6. 全検証を再実行
7. `DOCUMENTATION.md`へ変更前後の評価と根拠を記録
8. 全軸85点以上、Critical/Highバグ0、全必須検証成功まで継続

優先順位：

1. 進行不能
2. セーブ破損
3. モバイル操作不能
4. 読めない・理解できない
5. 性能
6. 視覚
7. 音
8. 小規模な追加コンテンツ
