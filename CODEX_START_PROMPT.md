/goal

このリポジトリで、ブラウザ探索アドベンチャーゲーム「月影町 ― 十三番目の回覧板」を、公開可能な完成品として設計・実装・検証・改善してください。

これは一度の長時間自律作業として任せます。短いプロトタイプ、画面モック、技術検証だけで終了せず、仕様に定義された7日間の物語、3Dの町の探索、転入届、住民票、自室、郵便受け、会話、手紙、掲示板、回覧板、セーブ、3種類以上のエンディング、PC・モバイル対応、自動テスト、E2E、PWA、ドキュメントまで完成させてください。

作業開始時に、次のファイルを必ずすべて読んでください。

- `AGENTS.md`
- `PRODUCT_SPEC.md`
- `STORY_BIBLE.md`
- `UX_VISUAL_SPEC.md`
- `ACCEPTANCE_TESTS.md`
- `PLAN.md`
- `DOCUMENTATION.md`

これらをプロジェクトの永続的な仕様・記憶として扱ってください。`PRODUCT_SPEC.md`を製品仕様のsource of truth、`AGENTS.md`を常時守る開発規則、`ACCEPTANCE_TESTS.md`を完成判定として扱います。

最初にリポジトリ、環境、既存コード、Git状態、利用可能なNode/npm/ブラウザテスト環境を調査してください。その結果を踏まえ、`PLAN.md`を具体的なマイルストーン、受入条件、検証コマンド付きで更新してください。計画を作った後は私の承認を待たず、すぐに実装を開始してください。通常の設計・実装判断は自分で行い、質問で作業を止めないでください。

各マイルストーンで、必ず次を繰り返してください。

1. 未完了の最上位要件を選ぶ
2. 実装する
3. lint、typecheck、unit test、buildなど必要な検証を実行する
4. 失敗があれば次へ進まず修正する
5. Playwright等で実画面を開く
6. デスクトップとモバイルのスクリーンショットを保存する
7. 文字切れ、重なり、操作、コンソール、物語進行、3Dの構図を点検する
8. 問題があれば修正して再検証する
9. `PLAN.md`と`DOCUMENTATION.md`へ進捗、判断、結果、再開点を書く
10. 次のマイルストーンへ進む

長時間作業中は会話コンテキストだけに依存しないでください。`PLAN.md`と`DOCUMENTATION.md`を外部記憶として継続更新してください。コンテキストが不足した場合は、仕様・計画・記録を再読し、テストで現状を確認し、未完了の最上位要件から自律的に再開してください。

重要制約：

- 外部の有料API、生成AI API、バックエンド、ログイン、広告、課金を導入しない
- Cloudflare Pagesで静的公開可能にする
- PCと一般的なスマートフォンのブラウザで遊べるようにする
- 3Dは広さより密度を優先した2.5D・低ポリゴン方式
- 権利不明の画像、音声、3Dモデル、フォントを使わない
- 素材は自作、プリミティブ3D、SVG、Canvas、Web Audio、または明確なライセンスのものに限定
- プレイヤー入力を安全に扱い、秘密情報や個人情報を外部送信しない
- 既存の未コミット変更やGit履歴を破壊しない
- force push、本番push、本番デプロイ、課金、不可逆操作は明示許可なしに行わない
- 必須機能未完成の間、写真モード、季節差分、管理画面、マルチプレイなどへ逃げない
- 高度だが不安定な構成より、動作と完成を優先する
- テストを通すために要件を無効化しない
- 進行不能、セーブ破損、モバイル操作不能を最優先で修正する

「完了」はコードを書いた時点ではありません。`PRODUCT_SPEC.md`の成果物、`ACCEPTANCE_TESTS.md`の受入条件、主要E2E、3エンディング、モバイル、セーブ復旧、ビルドを実際に検証した時点です。

必須条件が完成した後も作業余力がある場合は停止せず、次の品質改善ループを行ってください。

1. 全品質コマンドとE2Eを実行
2. デスクトップ・モバイルのスクリーンショットを確認
3. 完成度、初見理解、操作性、視覚的一貫性、物語、モバイル、性能、アクセシビリティ、テスト信頼性、保守性を100点で採点
4. 最低点の一項目を選ぶ
5. その項目を改善
6. 全検証を再実行
7. `DOCUMENTATION.md`へ変更前後の評価と根拠を書く
8. 全軸85点以上、Critical/Highバグ0、必須検証全成功まで繰り返す

追加機能を無秩序に増やすのではなく、実画面、操作、テスト、物語の理解度を基準に磨き込んでください。

最終的に以下を残してください。

- 完成したWebゲーム
- ソースコードと必要素材
- 起動・テスト・ビルド・Cloudflare Pages公開手順
- 単体・コンポーネント・E2Eテスト
- コンテンツ整合性検証
- デスクトップ・モバイルの主要スクリーンショット
- コンテンツ追加ガイド
- 依存・ライセンス情報
- 更新済み`PLAN.md`
- 完成状況、主要判断、テスト結果、既知問題、デモ手順、Git状態を含む`DOCUMENTATION.md`
- 実装したもの、未実装、検証結果、デプロイ可否の最終報告

今から仕様の調査、計画、実装、検証、改善まで連続して進めてください。
