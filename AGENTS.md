# AGENTS.md

## Mission

ブラウザ探索アドベンチャーゲーム「月影町 ― 十三番目の回覧板」を、PCとスマートフォンで最後まで遊べる公開可能な完成品として構築する。

優先順位：

1. 最初から3種類以上のエンディングまで進行不能なく遊べる
2. 月影町へ転入し、住民になったと感じられる
3. PCとスマートフォンの両方で操作できる
4. セーブ、復旧、テスト、ビルドが信頼できる
5. 世界観、3D、UI、音、文章が一貫している
6. コンテンツを追加しやすい

## Source of truth

1. `PRODUCT_SPEC.md`
2. `STORY_BIBLE.md`

矛盾時は上位を優先する。

## Working rules

- 原則として質問で作業を止めず、合理的な仮定を置いて進める。
- 必須要件を削って、見た目だけのデモにしない。
- 動かない高度な設計より、動く堅実な設計を選ぶ。
- TypeScriptの`any`は原則使用しない。
- 重要なゲームロジックを3D表示層から分離する。
- 会話、手紙、回覧板、イベント、クエストはデータ駆動にする。
- セーブデータにスキーマバージョンを持たせる。
- ユーザー入力をHTMLとして直接挿入しない。
- 権利不明の画像、音声、3Dモデル、フォントを使わない。
- 外部素材を使った場合は`THIRD_PARTY_NOTICES.md`へ出所、ライセンス、用途を書く。
- 既存の未コミット変更、Git履歴、ユーザーファイルを破壊しない。
- force push、本番公開、課金、秘密情報の外部送信はしない。
- UI文言とゲーム本文は日本語を基本とする。

## Required checks

節目ごとに、利用可能な範囲で以下を実行する。

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

主要体験の完成後：

```bash
npm run test:e2e
npm run quality
```

失敗を放置したまま次のマイルストーンへ進まない。

## Visual verification

見た目を伴う変更はブラウザで実際に確認する。

- デスクトップとモバイルの両方をPlaywright等で開く
- 主要画面をスクリーンショット保存
- 文字切れ、重なり、コントラスト、操作不能、ローディング停止を点検
- 3Dは夕方・夜・低画質を確認
- コンソールエラーを確認
- 問題があれば修正し再撮影する

## Persistent memory

長時間作業では会話コンテキストだけに依存しない。

各マイルストーン終了時に`PLAN.md`と`DOCUMENTATION.md`へ以下を書く。

- 完了したこと
- 検証結果
- 主要な判断と理由
- 未解決問題
- 次に行うこと
- 再開コマンド

コンテキスト不足時は、仕様、計画、記録を再読し、テストを実行して未完了の最上位要件から再開する。
